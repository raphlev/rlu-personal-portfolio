# Skill: CV to Portfolio (Word Document → portfolio.md)

## Purpose

Read an updated `Raphael_Leveque_CV.docx` and sync the changes back into
`public/portfolio.md`. This is the reverse of the portfolio-to-cv skill.

Use this after editing the CV in Word (e.g. to add a new job, update dates,
revise bullet points) and wanting those changes reflected in the portfolio
website source.

## Trigger phrases

- "sync my CV back to portfolio"
- "update portfolio from my Word CV"
- "I edited the CV, update portfolio.md"
- "reverse sync CV to portfolio.md"

## What the script syncs (and what it does NOT touch)

### Sections updated automatically

| portfolio.md section | What gets updated |
|----------------------|-------------------|
| `## timeline` | All jobs: period, company, role, location, subsections, bullets, project list |
| `## skills` | All skill groups + certifications JSON block |
| `## about` → education | Education entries only (bio paragraphs left untouched) |
| `## works` | Industry client grid (rebuilt from Notable Clients table in CV) |
| `## testimonials` | All testimonial quotes and attributions |

### Sections intentionally left unchanged

These sections have no equivalent in the CV and must be edited directly in
`portfolio.md`:

- `## intro` — greeting, badge, CTA labels
- `## navbar` — nav link labels
- `## services` — core expertise cards
- `## experience` — stat counters
- `## portfolio` — key engagement cards
- `## footer` — social links

## Step-by-step workflow

### 1. Confirm the CV is in the expected format

The script expects a CV generated by `generate_cv_complete.py` (or manually
edited while preserving that format). Key structural signals it relies on:

| Signal | How detected |
|--------|-------------|
| Section headings | All-uppercase paragraphs with a bottom border |
| Job headers | Bold text + `\t` + right-aligned date |
| Company lines | Italic first run, separated by `\|` from location |
| Subsection headings | Bold, mixed-case, < 120 chars, not a job header |
| Bullets | `List Bullet` paragraph style |
| Project bullets | `List Bullet` style with bold metadata header before `: ` and normal description after |
| Skills table | 2-column borderless table (first label is a skill group name) |
| Languages table | 2-column borderless table (first label matches known language names) |
| Notable Clients table | 2-column borderless table (first label matches known sector names, e.g. "Aerospace & Defense") |
| Testimonials | Bold attribution run (Name — Role, Company) followed by italic quote paragraph |

### 2. Dry-run first

Always run with `--dry-run` first to see what would change without writing:

```bash
cd D:\ClaudeWS\rlu-portfolio\cv_generator
python cv_to_portfolio.py --dry-run
```

Expected output:
```
Reading CV: Raphael_Leveque_CV.docx
Parsed:
  Name         : Raphael Leveque
  Jobs         : 7
  Education    : 2
  Certs        : 4
  Skill groups : ['PLM & Architecture', 'Cloud & IoT', ...]
  Summary paras: 3
  Notable clients sectors: 6
  Testimonials : 4
[DRY RUN] Changes would be written. ~N lines added, ~N lines removed
```

### 3. Apply the changes

```bash
cd D:\ClaudeWS\rlu-portfolio\cv_generator
python cv_to_portfolio.py
```

### 4. Review the diff in portfolio.md

After running, open `public/portfolio.md` and review the changed sections.
Pay particular attention to:

- Job periods — verify the format is "Month YYYY – Month YYYY" or "Month YYYY – Present"
- Company names — script preserves the legal suffixes (S.A.S., Inc., etc.)
- Skill items — check no items were lost or duplicated
- Certification dates — verify they parsed correctly

### 5. Rebuild the CV to confirm round-trip

Run the forward script to confirm the updated portfolio.md generates a
clean CV:

```bash
cd D:\ClaudeWS\rlu-portfolio\cv_generator
python generate_cv_complete.py
```

## What to do if the script misparses something

The script uses structural heuristics. If you see garbled output, the most
likely causes are:

| Symptom | Likely cause | Fix |
|---------|-------------|-----|
| Jobs missing | Section heading not detected | Check the heading is uppercase with a bottom border |
| Company name wrong | `\|` separator missing in company line | Re-generate the CV from portfolio.md and re-edit |
| Skills empty | Table not 2-column | Check the skills table has exactly 2 columns |
| Notable Clients not synced | First cell label not in `sector_labels` set | Ensure sector names match exactly (e.g. "Aerospace & Defense") |
| Testimonials missing | Attribution paragraph not bold | Re-generate the CV; testimonial bold formatting may have been stripped |
| Dates look odd | Month abbreviation not recognised | Spell out months fully in the CV |
| Extra blank lines | Spacer paragraphs parsed as content | These are filtered out (empty text paragraphs are skipped) |

If the script cannot parse your CV reliably, the safest workflow is to edit
`portfolio.md` directly and re-generate the CV. The script is a convenience
for when the CV is the primary edited artifact.

## Round-trip guarantee

The two scripts form a consistent pair:

```
portfolio.md  ──[generate_cv_complete.py]──►  Raphael_Leveque_CV.docx
portfolio.md  ◄──[cv_to_portfolio.py]───────  Raphael_Leveque_CV.docx
```

Running both in sequence (forward then reverse) should produce a `portfolio.md`
that is semantically equivalent to the original, with only whitespace or
ordering differences in the JSON blocks.

If you notice a field that doesn't round-trip correctly, update both scripts
together to keep them in sync.
